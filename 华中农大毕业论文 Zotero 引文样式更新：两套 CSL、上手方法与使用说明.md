# 华中农大毕业论文 Zotero 引文样式更新：两套 CSL、上手方法与使用说明

4 年前，那是我的本科毕业季，为了方便自己和大家管理参考文献，我手搓了一个适配华中农业大学毕业论文格式的 CSL，并写过一篇简单的使用说明。

4 年间陆续有同学在使用，也不断有人反馈一些边角问题，我也尽可能提供了帮助和进行了修复。

4 年后的现在，AI 大行其道，于是在AI的帮助下，我对照学校最新的学位（毕业）论文撰写规范，对样式做了一轮比较系统的修订。

这次主要做了两件事：

- 按学校规范，拆分为两套样式：
  - 自然科学类
  - 人文社科类
- 针对实际使用中最容易踩坑的地方，重新修正了文内引注和参考文献表的多处细节

如果你正好在写毕业论文，希望这篇能帮你少折腾一点格式问题。

------

## 一、这次更新了什么

这次更新，核心不是“换个文件名”，而是针对学校规范做了一轮更细的适配。

### 1. 拆分为两套 CSL

学校的毕业论文规范分为：

- 自然科学类
- 人文社科类

两者在文内引用规则上基本一致，但参考文献表中的若干细节并不完全相同，尤其是：

- 中文期刊论文格式
- 部分中文条目的标点组织
- 个别文献类型的著录细节

因此，这次不再强行用一个样式兼容所有情况，而是分别提供两套：

- [自然科学类 CSL](sandbox:/mnt/data/hzau-thesis-author-date-natural-sciences-v3.csl)
- [人文社科类 CSL](sandbox:/mnt/data/hzau-thesis-author-date-social-sciences-v3.csl)

### 2. 修正了文内引注中的作者显示问题

旧版本和早期修改版里，比较典型的问题包括：

- 中文两位作者之间多了逗号
- 英文两位作者显示不够规范
- 中英文混合引注时，括号和分隔符容易乱掉
- 同一括号内多篇文献的排序和分隔不够稳定

这次更新后，主要改成了：

- 中文 1 位作者：`（张三 2024）`
- 中文 2 位作者：`（张三和李四 2024）`
- 中文 3 位及以上作者：`（张三等 2024）`
- 英文 1 位作者：`（Smith 2024）`
- 英文 2 位作者：`（Smith and Jones 2024）`
- 英文 3 位及以上作者：`（Smith et al 2024）`

同时，同一括号内多篇文献会尽量按年份顺序排列。

### 3. 修正了作者人数截断规则

学校要求：

- 作者人数在 20 人及以下时全部列出
- 超过 20 人时，列前 20 人，后加“等”或 `et al`

这次已经按这个逻辑调整。

### 4. 修正了学位论文等特殊文献类型的显示

这次重点处理了几类特别容易出错的文献：

- 学位论文
- 专利
- 网络文献
- 专著章节
- 会议论文集
- 会议论文

例如中文学位论文，现在会尽量按：

```
作者．题名．[博士学位论文]．地点：单位，年份．
```

这样的结构输出，而不再是早期那种不完整或标签不规范的形式。

### 5. 优化了英文期刊简称输出

学校要求外文刊名缩写后不加缩点。
这次样式已尽量按这个方向处理，例如：

- `Nat Genet`
- `Nat Commun`
- `Eur J Plant Pathol`

而不是带一串缩点的形式。

### 6. 自然科学类样式默认去掉期号

自然科学类学校示例更接近：

```
刊名，年份，卷号：页码
```

因此自然科学类样式里，期刊论文默认更倾向于不显示期号，以尽量贴近规范示例。

## 二、如何上手使用

如果你之前没有用过 Zotero，也完全可以直接从这里开始。

### 第一步：安装 Zotero

先去官网下载并安装 Zotero。
安装完成后，建议同时安装浏览器插件，方便你从网页、数据库、期刊页面一键抓取文献信息。

### 第二步：下载对应的 CSL 文件

根据你的论文类别下载对应样式：

- [自然科学类 CSL](sandbox:/mnt/data/hzau-thesis-author-date-natural-sciences-v3.csl)
- [人文社科类 CSL](sandbox:/mnt/data/hzau-thesis-author-date-social-sciences-v3.csl)

### 第三步：导入 CSL 到 Zotero

在 Zotero 中依次打开：

```
编辑` → `设置` → `引用` → `样式
```

然后点击：

```
+` 或 `添加样式
```

选择刚才下载的 `.csl` 文件导入即可。

### 第四步：在 Word 中调用样式

如果你已经安装了 Zotero 的 Word 插件，那么在 Word 中会看到 Zotero 工具栏。
写论文时，可以直接：

- 插入引文
- 添加脚注
- 生成参考文献表
- 一键切换引文样式

选择刚才导入的样式即可。

## 三、注意事项

这里是最重要的部分。因为 CSL 可以解决很多问题，但它不是万能的。

### 3.1 有些事情，CSL 做不到

#### 1）中文参考文献按拼音排序

学校要求中文参考文献按第一作者姓名拼音排序。
这一点，**Zotero + CSL 目前无法稳定、可靠地自动实现**。

**最终提交论文前，请务必人工检查中文参考文献的排序。**

#### 2）脏数据、错误数据，样式也救不了

如果你的文献条目本身就录错了，例如：

- 作者姓名拆分错误
- 标题里混入 HTML 标签
- 年份缺失
- 页码缺失
- 中文标点和英文标点混乱

那么再好的 CSL 也只能“尽量少出错”，不能替你把错误数据自动修正成正确格式。

------

### 3.2 有些信息，需要你自己补全

这一部分非常关键。
下面我尽量按 **Zotero 中文界面的实际字段名** 来写，方便大家直接对照填写。

#### 1）语言（Language）

适用场景：

- 区分中文文献和英文文献的排版规则

建议：

- 英文文献填写：`en` 或 `English`
- 中文文献可留空，或填写：`zh-CN`

如果英文条目没有正确标注语言，可能会被按中文规则处理。

#### 2）刊名简称（Journal Abbr.）

适用场景：

- 外文期刊论文

学校要求外文期刊名使用缩写，因此建议英文期刊条目尽量补全这个字段。
否则有时只能输出期刊全称，或输出不够规范的简称。

#### 3）地点（Place）

适用场景：

- 图书
- 学位论文
- 会议论文集
- 标准等

例如学位论文中，学校要求通常是：

```
地点：单位，年份
```

如果缺少“地点”，输出往往会不完整。

#### 4）出版社（Publisher）

适用场景：

- 图书
- 标准
- 部分论文集
- 学位论文中的保存单位/授予单位等

这一项经常和“地点”一起使用。

#### 5）日期（Date）

适用场景：

- 所有文献，尤其是：
  - 期刊论文
  - 网络文献
  - 专利
  - 学位论文

如果日期缺失，就可能出现：

- 没有年份
- 文内引注出现 `[n.d.]`
- 同作者同年排序异常

#### 6）卷次（Volume）

适用场景：

- 期刊论文

如果卷次缺失，而条目里只有期号，输出会显得不完整。
这一点在中文期刊中比较常见。

#### 7）期号（Issue）

适用场景：

- 期刊论文

自然科学类样式里默认更倾向于不显示期号，但这不代表这个字段可以随便空着。
如果你的条目本来就卷、期信息不全，最终仍然可能影响输出质量。

#### 8）页码（Pages）

适用场景：

- 期刊论文
- 图书章节
- 会议论文
- 专利等

页码缺失时，参考文献通常一眼就会显得不完整。

#### 9）网址（URL）

适用场景：

- 网络文献

网络文献如果没有网址，基本就不成立了。

#### 10）访问时间（Accessed）

适用场景：

- 需要记录访问日期的网络文献

不是所有人都会用到，但如果你引用的是网页内容，建议一并完善。

#### 11）标题（Title）

适用场景：

- 所有条目

有些从网页或数据库抓取的题名中，会混入奇怪内容，例如 HTML 标签、错误空格、大小写混乱等，这些都需要手动检查。

#### 12）出版物（Publication Title）

适用场景：

- 期刊名
- 会议名
- 文集名等

如果这里填得不规范，最终参考文献中的刊名、会议名就会跟着出问题。

#### 13）作者（Author）

适用场景：

- 所有条目，尤其是英文文献

这是一项经常被忽略、但实际影响很大的字段。
例如本该输出成：

```
Dangl JL
```

结果却变成：

```
L Dangl J
```

或者本该输出成：

```
Shi Y
```

结果却变成：

```
Shi Yajuan
```

大多数情况下，都是作者字段录入不规范造成的。

## 四、为什么推荐大家使用 Zotero 来管理参考文献

说实话，很多同学在论文最后阶段最头疼的，不是实验、不是写作，而是参考文献。

常见痛苦包括：

- 文内引用一个个手敲
- 换期刊或换学校格式时整篇重改
- 英文作者缩写规则搞不清
- 中英文混排容易出错
- 改了一条，后面全乱
- 最后查重、送审、提交前还要通宵整理格式

这些事情，本质上并不值得你花那么多时间。

### 1. Zotero 能把“文献收集”和“文献引用”串起来

很多人最大的问题是：

- 文献下载了一堆
- PDF 存了一堆
- 最后写论文时找不到
- 找到了又不知道该怎么引用

Zotero 的优势在于，它把这些动作放在同一个系统里：

- 抓取文献信息
- 管理 PDF
- 分类整理
- 记录标签
- 插入引文
- 生成参考文献表

这样你的文献管理不会和写作完全脱节。

### 2. 它能大幅降低格式返工成本

只要条目录入相对规范、样式设置正确，Zotero 可以帮你完成绝大多数重复性工作。

比如：

- 插入引文
- 自动更新参考文献表
- 切换引文样式
- 统一作者显示规则
- 统一年卷页格式

这比手动维护参考文献高效太多，也更不容易漏改。

### 3. Zotero 的生态成熟，适合长期使用

它不只是“毕业论文临时工具”，更适合作为长期科研工具。
无论你以后是继续读研、读博，还是写综述、写基金、投期刊，都可以一直沿用。

你在 Zotero 中沉淀下来的，不只是一个文献库，更是一套相对稳定的科研工作流。

### 4. 对中文用户也越来越友好

虽然很多教程是英文的，但 Zotero 本身并不难上手。
而且只要样式文件准备好，中文论文写作同样可以受益很大。

与其在最后阶段反复手改参考文献，不如从一开始就把文献管理交给更适合做这件事的工具。

------

## 五、最后再提醒几句

这次更新后的样式，已经尽量按照学校规范去适配，并对一些最容易出问题的细节做了修正。

**但它并不是“导入即完美”的魔法文件**，真正决定最终输出质量的，仍然包括两件事：

- 你是否选择了正确的样式版本
- 你的 Zotero 条目录入是否规范完整

所以比较稳妥的做法是：

1. 先导入对应的 CSL
2. 检查并补全关键信息
3. 最后人工复核一次参考文献排序和个别特殊条目

这样基本就能省下很多后期返工时间。
